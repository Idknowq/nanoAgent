为什么并行缓存命中率低
1. 子 Agent 冷启动：每个子 Agent 都是全新的独立上下文，第一次调用缓存命中接近 0%。一个子 Agent 生命周期只有 3-11 次调用，还没摊销冷启动成本就结束了。
2. 主 Agent 上下文震荡：每返回一个子 Agent 结果（不同包的审计报告），主 Agent 上下文结构就变一次，缓存被打断重构（#7 从 86% 跌到 9%）。
